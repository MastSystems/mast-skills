## Tripwires table T1–T7 (D-absorb-tripwires) — VERBATIM

> Deviate only on these.

| # | condition | action |
|---|---|---|
| T1 | Top queue item blocked by unresolved dependency markers | write steering note via `mast loop event steering`, move to Skipped, stop. No code commit. |
| T2 | Project's test-cmd fails on HEAD *before* your changes | steering note, commit only plan.md, stop |
| T3 | Project's lint-cmd reports an error on a work-unit NOT your target | same as T2 |
| T4 | ≥4 open steering notes accumulated | insert meta-directive "queue frozen until notes clear", commit plan.md, stop |
| T5 | You want to emit the completion sentinel but corpus count is not zero | do NOT emit. Finish mode normally. |
| T6 | Any git commit in last 5 not made by the loop | steering note "external commit in scope", continue mode |
| T7 | Corpus count command returns zero | emit the completion sentinel, stop. Valid from any mode. |

## Invariants / commit-prefix strings (C4) — VERBATIM (do not change these strings)

- inspector commit prefix: `chore(loop): inspector report`
- build commit prefix: per RALPH_LOOP.md's graduation-commit-pattern
- escalated-graduation suffix: `[escalated]` appended to the build commit subject
- escalate request prefix: `chore(loop): escalate <work-unit-id>`
- escalate failure prefix: `chore(loop): escalate-failed <work-unit-id>`
- curate commit prefix: `chore(loop): curate`
- compact commit prefix: `chore(loop): rebuild patterns after compaction`
- populate-plan commit prefix: `chore(loop): populate plan queue` (legacy; new kickoffs use strategize)
- strategize commit prefix: `chore(loop): strategize`
- re-strategize request prefix: `chore(loop): request re-strategize`
- completion sentinel: `<promise>COMPLETE</promise>` (MUST be emitted on a line by itself — run.sh verifies the line-anchored form AND re-queries the corpus count before exiting; a sentinel quoted inside prose will not trigger termination)
- reopen marker: `(reopened by inspector xN)` appended to a queue line
- one-commit invariant: each build-mode iteration produces exactly one git commit between its `IterStart` and `IterEnd` events; multi-commit iterations are a discipline violation surfaced by inspect-mode audits and by parsing of chore commit prefixes

## Completion sentinel + reopen marker (C5) — VERBATIM

- **completion sentinel:** `<promise>COMPLETE</promise>`
  - MUST be emitted on a **line by itself** — `run.sh` verifies the line-anchored
    form AND re-queries the corpus count before exiting; a sentinel quoted inside
    prose will not trigger termination.
  - **Never quote the completion sentinel in prose or markdown.** Refer to it as
    "the completion sentinel".
- **reopen marker:** `(reopened by inspector xN)` appended to a queue line.

## Loop state filenames (C9) — VERBATIM

The loop's event-sourced state and artifact files:

- `.mast/loop/plan.md` — regenerated by the Rust runner from `events.jsonl`
- `events.jsonl` (`.mast/loop/events.jsonl`) — the durable event stream;
  `plan.md` and `progress.md` are projections of it
- `progress.md` — regenerated from `events.jsonl` (`## Inspector Reports`,
  `## Codebase Patterns`, `## Session Log`)
- `.mast/loop-<plan>.md` — per-plan iteration protocol file (loop handoff)
- `logs/progress-archive-*.md` — compaction archives
- `RALPH_LOOP.md` — the loop's editing-instructions / graduation-commit-pattern source
